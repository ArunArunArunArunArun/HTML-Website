Uniform Resource Locator

A Uniform Resource Locator, more commonly known by the the acronym 'URL', is the adress of a resource on the web.

    'Https' is the protocol for the url. 'https' is a protocol for transmitting HTML between web browsers and web servers. This is the core protocol that governs transmission of data on the internet. It is an access protocol. The client computer requests through HTTP that the website be available. The web server responds via HTTP and shares the website data.

    Next is the colon and double slash. The colon simply separates the protocol and domain/subdomain. The double slash was a mistake which occurred when the web was first created. It has no real purpose.

    A subdomain is a way to organise a website, it is an additional part of a domain and is used to navigate subsections of a website. For example, louisthechild.com has a merch shop, acessed by adding the subdomain "store" (store.louisthechild.com)

    'www' is an acronym for 'World Wide Web'.

    The domain is the name of the website.

    The root domain is the structure of the URL. Examples of a root domain are:
    • .com
    • .net
    • .gov

    The path to the file comes after the root domain. It navigates you to the exact location of the webpage.

    The anchor is the page number or another piece of clickable text on a website.

    Finally, the URL query parameters are pieces of information about the person on the website. The parameter comes after a question mark. The parameter is in the form "Variable" = "Value".
